# Nebula Browser v0.4.0 — Design Document

> **Status**: Planning
> **Target**: Late July 2026
> **Theme**: Community, Personalization, and Accounts

This document outlines the four major features planned for v0.4.0. Each section covers the architecture, technology choices, and implementation plan.

## Feature 1: Sign-in & Data Sync (Supabase)

### Why Supabase?

After evaluating Clerk, Firebase, Auth0, and Pocketbase, **Supabase** is the recommended choice:

| Criteria | Supabase | Firebase | Clerk | Pocketbase |
|---|---|---|---|---|
| Free tier | 500MB DB, 50K users, 1GB storage | 1GB DB, 10K users | 10K users (no DB) | Self-hosted only |
| Database | PostgreSQL (SQL) | Firestore (NoSQL) | None (bring your own) | SQLite |
| Auth methods | Email, OAuth, magic links, magic URLs | Email, OAuth, phone | Email, OAuth, SSO | Email, OAuth |
| Storage | Built-in (S3-compatible) | Cloud Storage | None | Built-in |
| Realtime | Built-in (Postgres CDC) | Built-in | None | None |
| Open source | Yes (self-hostable) | No | No | Yes |
| Next.js integration | Excellent (@supabase/ssr) | Good | Excellent | Manual |
| Vendor lock-in | Low (Postgres + open source) | High | Medium | Low |
| Price at scale | $25/mo for 8GB | Pay-per-use | $25/mo for 1K users | Free (self-hosted) |

**Supabase wins because:**
1. PostgreSQL — we're already using Prisma, can migrate seamlessly
2. Free tier covers a small browser project for months
3. Built-in storage for wallpapers, plugins, avatars
4. Realtime sync for tabs/bookmarks across devices
5. Open source — can self-host later if needed
6. Row-level security (RLS) — per-user data isolation built into the DB

### What gets synced

| Data type | Synced? | Storage | Realtime? |
|---|---|---|---|
| Bookmarks | ✅ | Postgres table | Yes |
| Browsing history | ✅ | Postgres table | No (batch) |
| Settings (accent, theme, glass) | ✅ | Postgres table | Yes |
| AI conversations | ✅ | Postgres table | No |
| Open tabs | ✅ (optional) | Postgres table | Yes |
| Downloads | ❌ | Local only | N/A |
| Passwords | ❌ | (future: encrypted) | N/A |

### Auth methods

- **Email + password** (traditional)
- **Google OAuth** (one-click sign-in)
- **GitHub OAuth** (for developers)
- **Magic link** (passwordless email)

### Database schema (Prisma)

```prisma
model User {
  id           String   @id @default(cuid())
  email        String   @unique
  name         String?
  avatarUrl    String?
  createdAt    DateTime @default(now())
  bookmarks    Bookmark[]
  history      HistoryEntry[]
  settings     Settings?
  conversations Conversation[]
  devices      Device[]
}

model Bookmark {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  title     String
  url       String
  favicon   String?
  order     Int      @default(0)
  createdAt DateTime @default(now())
}

model HistoryEntry {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  title     String
  url       String
  visitedAt DateTime @default(now())
}

model Settings {
  userId    String   @id
  user      User     @relation(fields: [userId], references: [id])
  accent    String   @default("cyan")
  theme     String   @default("dark")
  glass     String   @default("strong")
  wallpaper String   @default("aurora")
  updatedAt DateTime @updatedAt
}

model Device {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  name      String   // "MacBook Pro", "Windows PC", etc.
  lastActive DateTime @default(now())
  openTabs  Json     // array of { url, title }
}
```

### Implementation plan

1. Create Supabase project at supabase.com (free)
2. Add `@supabase/supabase-js` and `@supabase/ssr` to the project
3. Set up environment variables: `SUPABASE_URL`, `SUPABASE_ANON_KEY`
4. Create Prisma schema + run migration
5. Build sign-in UI (email + Google OAuth buttons)
6. Add "Sign in" button to settings panel
7. Wire bookmark/history/settings stores to sync on change
8. Add "tabs from other devices" panel
9. Add sign-out option

## Feature 2: Wallpaper Marketplace

### Overview

Community-created wallpapers that users can browse, preview, and apply — like Opera GX's wallpaper system. Supports both static (PNG/JPG) and animated (MP4/WebM) wallpapers.

### Wallpaper types

| Type | Format | Max size | Notes |
|---|---|---|---|
| Static | PNG, JPG, WebP | 5MB | Single image |
| Animated | MP4, WebM | 20MB | Short loop (5-15s recommended) |
| Gradient | CSS | N/A | Code-defined, no file |
| Live | CSS + JS | 50KB | Animated CSS/JS scene |

### Where wallpapers apply

1. **New Tab Page background** — behind the logo and quick links
2. **Browser chrome tint** (optional) — subtle color wash on the toolbar/tab bar
3. **Full background** (Opera GX mode) — wallpaper visible behind everything with glass surfaces

### Upload flow

1. User clicks "Upload"
2. Selects file (PNG/JPG/MP4/WebM) or picks a gradient
3. Adds title, description, tags
4. Preview shows how it'll look
5. Submit → uploaded to Supabase Storage
6. Moderation queue (auto-scan + manual review for public)
7. Published to marketplace

## Feature 3: Plugin System

### Overview

A sandboxed plugin system that lets the community extend Nebula Browser with custom features — themes, sidebar panels, toolbar buttons, AI enhancements, and content modifiers.

### Plugin types

| Type | What it does | Example |
|---|---|---|
| **Theme** | Custom CSS, accents, fonts, glass effects | "Cyberpunk Theme" — neon pink/blue everywhere |
| **Sidebar Panel** | Adds a custom panel to the AI sidebar | "Weather Panel" — shows local forecast |
| **Toolbar Action** | Adds a button to the toolbar | "Screenshot" — capture the current page |
| **AI Extension** | Custom prompts, modes, or providers | "Code Reviewer" — specialized code analysis |
| **Content Modifier** | Modifies page content after load | "Ad Blocker" — removes ad elements |
| **New Tab Widget** | Adds widgets to the new tab page | "Clock Widget" — analog clock on NTP |

### Plugin manifest (manifest.json)

```json
{
  "name": "Weather Panel",
  "id": "weather-panel",
  "version": "1.0.0",
  "author": "jane_doe",
  "description": "Shows local weather forecast in a sidebar panel",
  "type": "sidebar-panel",
  "permissions": [
    "network:fetch",
    "storage:local"
  ],
  "entry": "index.js",
  "styles": "styles.css",
  "config": {
    "defaultCity": "San Francisco",
    "units": "imperial"
  },
  "minAppVersion": "0.4.0"
}
```

### Plugin API (window.nebula)

```typescript
interface NebulaPluginAPI {
  // Theme plugins
  theme: {
    setAccent(color: string): void;
    setCSS(css: string): void;
    setBackground(url: string): void;
  };

  // Sidebar plugins
  sidebar: {
    registerPanel(id: string, component: React.FC): void;
    setActive(id: string): void;
  };

  // Toolbar plugins
  toolbar: {
    addButton(config: { id: string; icon: string; onClick: () => void; title: string }): void;
  };

  // AI plugins
  ai: {
    registerMode(id: string, config: { label: string; systemPrompt: string }): void;
    onMessage(callback: (msg: string) => void): void;
  };

  // Content plugins (run in webview context)
  content: {
    onPageLoad(callback: (url: string) => void): void;
    injectCSS(css: string): void;
    injectScript(js: string): void;
  };

  // Storage (sandboxed per-plugin)
  storage: {
    get(key: string): Promise<any>;
    set(key: string, value: any): Promise<void>;
  };

  // Network (requires permission)
  network: {
    fetch(url: string, opts?: RequestInit): Promise<Response>;
  };

  // Browser state (read-only)
  browser: {
    getCurrentTab(): { url: string; title: string };
    getBookmarks(): Bookmark[];
    getHistory(): HistoryEntry[];
  };
}
```

### Sandboxing

Plugins run in a **Web Worker** with limited access to the DOM:
- **Theme/toolbar/sidebar plugins** — run in the main renderer, but with a restricted API
- **Content plugins** — injected into webviews via `<script>` tag with Content Security Policy
- **No direct access** to `window`, `document`, or Electron APIs
- All actions go through the `window.nebula` API which validates permissions
