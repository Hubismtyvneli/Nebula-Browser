## Feature 1: Owned Content System (Wallpapers + Themes + Plugins)

### Overview
Wallpapers, themes, and plugins are no longer just "applied" вҖ” they're **owned**. Users browse the marketplace, click "Get" (free or paid), and the item appears in their **Profile library**. They can then switch between owned items from the profile or settings.

### Data model changes

```sql
-- New table: user_owned_content
-- Tracks which wallpapers/themes/plugins a user has acquired
CREATE TABLE public.user_owned_content (
  id          TEXT PRIMARY KEY,
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  content_id  TEXT NOT NULL,  -- wallpaper ID, theme ID, or plugin ID
  content_type TEXT NOT NULL,  -- 'wallpaper' | 'theme' | 'plugin'
  price_paid  NUMERIC(10,2) DEFAULT 0,  -- 0 = free
  acquired_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_id, content_id, content_type)
);

-- Add pricing + visibility fields to existing tables
ALTER TABLE public.wallpapers ADD COLUMN IF NOT EXISTS price NUMERIC(10,2) DEFAULT 0;
ALTER TABLE public.wallpapers ADD COLUMN IF NOT EXISTS visibility TEXT DEFAULT 'public'; -- 'public' | 'private' | 'unlisted'

ALTER TABLE public.plugins ADD COLUMN IF NOT EXISTS price NUMERIC(10,2) DEFAULT 0;
ALTER TABLE public.plugins ADD COLUMN IF NOT EXISTS visibility TEXT DEFAULT 'public';

-- New table: themes (distinct from wallpapers вҖ” full visual overhaul)
CREATE TABLE public.themes (
  id           TEXT PRIMARY KEY,
  author_id    UUID REFERENCES auth.users(id) ON DELETE SET NULL,
  author_name  TEXT,
  name         TEXT NOT NULL,
  description  TEXT,
  version      TEXT DEFAULT '1.0.0',
  icon         TEXT,
  -- Theme config stored as JSON
  config       JSONB NOT NULL,
  -- config includes: accent, glassIntensity, wallpaperId, fontFamily, borderRadius, customCSS, etc.
  tags         TEXT[] DEFAULT '{}',
  installs     INTEGER DEFAULT 0,
  rating       NUMERIC(2,1) DEFAULT 0,
  rating_count INTEGER DEFAULT 0,
  price        NUMERIC(10,2) DEFAULT 0,
  visibility   TEXT DEFAULT 'public',
  is_builtin   BOOLEAN DEFAULT FALSE,
  created_at   TIMESTAMPTZ DEFAULT now()
);

-- Enable RLS on all new tables
ALTER TABLE public.user_owned_content ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.themes ENABLE ROW LEVEL SECURITY;

-- Policies: users can only see/edit their own owned content
CREATE POLICY "Users view own owned content" ON public.user_owned_content
  FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users insert own owned content" ON public.user_owned_content
  FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users delete own owned content" ON public.user_owned_content
  FOR DELETE USING (auth.uid() = user_id);

-- Themes: public viewable, owner-editable
CREATE POLICY "Anyone views public themes" ON public.themes
  FOR SELECT USING (visibility = 'public');
CREATE POLICY "Users insert themes" ON public.themes
  FOR INSERT WITH CHECK (auth.uid() = author_id);
CREATE POLICY "Users update own themes" ON public.themes
  FOR UPDATE USING (auth.uid() = author_id);
CREATE POLICY "Users delete own themes" ON public.themes
  FOR DELETE USING (auth.uid() = author_id);
```

A new `nebula://profile` tab showing:
- **My Wallpapers** вҖ” grid of owned wallpapers (applied + unapplied)
- **My Themes** вҖ” list of owned themes
- **My Plugins** вҖ” list of owned/enabled plugins
- **Created by me** вҖ” wallpapers/themes/plugins the user has uploaded
- Each item has: Apply / Remove / Edit (if owner)

### Marketplace flow
1. Browse marketplace вҶ’ see price badge (Free / $2.99)
2. Click item вҶ’ detail page with preview, description, author, ratings
3. Click "Get" (free) or "Buy" (paid)
4. Item appears in Profile library
5. Apply from library or directly from marketplace

## Feature 3: Theme System (All-in-One)

### Overview
A "Theme" is a complete visual package that bundles:
- Accent color
- Glass intensity
- Active wallpaper
- Font family
- Border radius
- Custom CSS overrides
- Dark/light mode preference

Users can create, share, and apply themes вҖ” one click changes the entire browser look.

### Theme config schema (JSONB)
```typescript
interface ThemeConfig {
  accent: AccentName;
  glassIntensity: GlassIntensity;
  wallpaperId: string | null;
  fontFamily: string;          // "Inter", "JetBrains Mono", etc.
  borderRadius: number;        // 0-20px
  customCSS: string;           // user-authored CSS overrides
  titleBarStyle: "hidden" | "transparent";
  ntpLayout: "centered" | "grid" | "minimal";
  sidebarWidth: number;
}
```

### Theme marketplace
- Browse themes by category (Minimal, Neon, Dark, Light, Anime, Retro)
- Preview shows a mockup of the browser with the theme applied
- "Apply Theme" button вҖ” instantly changes accent + glass + wallpaper + fonts
- "Save current setup as Theme" вҖ” captures user's current settings into a theme

## Feature 4: Wallpaper Creator

### Overview
When uploading a wallpaper, users get a **creation flow** with options:

### Creation flow
1. **Upload file** вҖ” PNG, JPG, WebP, MP4, WebM
2. **Set metadata**:
   - Title
   - Description
   - Tags
   - Category (Abstract, Nature, Anime, Minimal, Neon, etc.)
3. **Visibility**:
   - **Public** вҖ” appears in marketplace for everyone
   - **Unlisted** вҖ” only accessible via direct link
   - **Private** вҖ” only visible to you (in your profile)
4. **Pricing**:
   - Free
   - Paid ($0.99 - $99.99)
5. **Preview** вҖ” see how it looks on the NTP
6. **Publish** вҖ” uploads to Supabase Storage + creates marketplace listing

### Creator dashboard (`nebula://creator`)
- My published wallpapers/themes/plugins
- Download counts, ratings, revenue
- Edit listing
- Delete/unpublish

## Feature 5: AI Page Awareness

### Overview
The AI assistant becomes deeply aware of the page the user is viewing вҖ” it can read page content, extract text, analyze structure, and answer questions about what's on screen.

### How it works
1. When a webview loads a page, inject a content script that:
   - Extracts the page's visible text (main content, not boilerplate)
   - Extracts headings, links, images, forms
   - Extracts meta tags (title, description, og:tags)
   - Sends this back to the AI sidebar via IPC
2. The AI system prompt is augmented with:
   - Page URL
   - Page title
   - Page description
   - First 2000 chars of visible text
   - Page structure (heading outline)
3. User can ask: "Summarize this page", "What's the main argument?", "Find the pricing section", "Translate the key points"

### New AI capabilities
- **"Read this page"** button in the AI sidebar вҖ” injects current page content into context
- **Auto-context** вҖ” when on a web page, AI automatically knows what you're viewing
- **Page Q&A** вҖ” "Does this page mention X?", "What's the author's conclusion?"
- **Smart suggestions** вҖ” based on page type (article вҶ’ summarize, code вҶ’ explain, product вҶ’ compare)

### Implementation
- `electron/main.js`: inject content script into webviews via `contents.executeJavaScript()`
- Content script extracts: `document.body.innerText.slice(0, 4000)`, headings, links, meta
- Sent to renderer via IPC вҶ’ stored in AI store as `currentPageContext`
- AI API route receives `context.text` in the request body
- System prompt includes: "The user is viewing: [URL] вҖ” [title] вҖ” [first 2000 chars of content]"
